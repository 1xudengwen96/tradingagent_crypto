# 加密货币多智能体交易系统 — 使用说明书

## 1. 系统简介

本系统是一个基于 LLM 多智能体架构的**日线级别**加密货币永续合约交易系统。

### 核心理念
- **铁三角架构**：技术分析师（价格行为） + 宏观分析师（资金费率/OI） + 组合管理者（ATR 硬编码仓位）
- **LLM 只定方向，数学管仓位**：大模型仅输出做多/做空/平仓方向，仓位大小由 Python 硬编码数学模型计算
- **UTC 00:00 日线对齐**：所有技术指标基于已收盘的日线 K 线，杜绝"半截子 K 线"假信号

## 3. 快速开始

### 3.1 基本用法

```python
from tradingagents.graph.crypto_trading_graph import CryptoTradingAgentsGraph

# 创建实例（不开启自动下单，仅分析）
graph = CryptoTradingAgentsGraph(
    debug=False,
    auto_execute=False,  # 先设为 False 看分析结果
)

# 运行分析
final_state, decision_text = graph.propagate("BTC/USDT:USDT")
print(decision_text)
```

### 3.2 自动执行交易

```python
# 开启自动下单（⚠️ 确保 sandbox_mode=False 时有真实资金风险意识）
graph = CryptoTradingAgentsGraph(
    auto_execute=True,   # 自动执行订单
    debug=True,          # 打印中间过程
)

final_state, decision_text, execution_result = graph.run("BTC/USDT:USDT")

if execution_result and execution_result.success:
    print(f"订单成功: {[o.get('id') for o in execution_result.orders]}")
else:
    print(f"订单失败: {execution_result.error}")
```

### 3.3 分析 + 手动确认后执行

```python

## 4. 风控规则详解

### 4.1 ATR 仓位计算公式

系统不使用 LLM 决定仓位，而是用硬编码数学模型：

```
止损距离 = 1.5 × ATR14（真实波动幅度）
基础仓位 = (总资金 × 1%) / (止损距离 / 入场价)
调整后仓位 = 基础仓位 × min(max(信念分/10, 0.5), 1.0)
最终仓位 = min(调整后仓位, 总资金 × 30%)
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 单笔最大亏损 | 1% | 无论信念分多高，单笔最多亏总资金 1% |
| ATR 倍数 | 1.5× | 止损 = 1.5 倍 ATR |
| 止盈1 | 2R | 止损距离的 2 倍 |
| 止盈2 | 3.5R | 止损距离的 3.5 倍 |
| 单笔上限 | 30% | 单笔仓位不超过总资金 30% |
| LITE 缩放 | 50% | LONG-LITE/SHORT-LITE 时仓位减半 |

### 4.2 BTC 200 日均线拦截器

```
如果 BTC 价格 < BTC 200 日均线 且 LLM 方向为 LONG/LONG-LITE：
    强制覆盖方向为 CLOSE（只能平仓，禁止开多）
```

这是硬性规则，LLM 无法覆盖。

### 4.3 止损/止盈强制挂单

每笔开仓订单发出后，系统**必须**立即挂出：
- **止损单（Stop Market）**：触发价格到达止损价时市价平仓
- **止盈单 1（Limit）**：挂 50% 仓位在 2R 止盈
- **止盈单 2（Limit）**：挂剩余 50% 仓位在 3.5R 止盈

如果信号中缺少止损/止盈价格，系统会自动用 ATR 回退计算。

## 4. Risk Parameters (Hardcoded Math — Cannot Be Overridden by LLM)

- Stop Distance: 2250.0 USDT
- Conviction Score: 7/10
- Risk Per Trade: 1.0%
```

## 6. 调度运行

### 6.1 推荐运行时间

**UTC 00:05**（北京时间 08:05）。此时日线已收盘定格，基于"既成事实"交易。

### 6.2 Cron 示例

```bash
# 每天 UTC 00:05 运行
5 0 * * * cd /home/forwhat/tradingagent_crypto && python3 run.py
```

### 6.3 示例运行脚本

```python
#!/usr/bin/env python3
"""每日自动分析并执行交易"""
import logging
from tradingagents.graph.crypto_trading_graph import CryptoTradingAgentsGraph

logging.basicConfig(level=logging.INFO)

config = {
    "auto_execute": True,      # 设为 True 自动下单
    "sandbox_mode": True,      # 沙盒模式
    "capital_usdt": 1000.0,
}

graph = CryptoTradingAgentsGraph(
    config=config,
    debug=False,
)

# 遍历交易对
for symbol in ["BTC/USDT:USDT", "ETH/USDT:USDT"]:
    _, decision, result = graph.run(symbol)
    logging.info("Symbol: %s | Decision: %s", symbol, decision[:50])
    if result:
        logging.info("Result: success=%s, orders=%s", result.success, len(result.orders))
```

### 9.2 风险控制

| 风险 | 保护措施 |
|------|----------|
| 黑天鹅暴跌 | 止损单自动挂出 + BTC 200MA 拦截 |
| LLM 幻觉 | 仓位由数学模型计算，LLM 不碰数字 |
| 半截 K 线假信号 | 强制过滤未收盘日线 |
| 过度集中 | 单笔仓位上限 30% 总资金 |
| 连续亏损 | 单笔最多亏 1%，10 次连亏最多亏 10% |
