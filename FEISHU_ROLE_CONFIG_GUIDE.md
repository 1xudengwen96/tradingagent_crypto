## 角色列表与分析流程

### 8 个角色及其分析职责

| 序号 | 角色 | Emoji | 分析内容 | 数据来源 | 输出要求 |
|------|------|-------|----------|----------|----------|
| 1 | 📈 **市场分析师** | 📈 | 技术面分析（K 线、均线、MACD、RSI、布林带等） | Bitget OHLCV 数据 | 必须包含具体数值和信号 |
| 2 | 😊 **情绪分析师** | 😊 | 市场情绪分析（资金费率、持仓量、订单簿深度） | Bitget 资金费率/持仓量/订单簿 | 必须引用真实数据 |
| 3 | 📰 **新闻分析师** | 📰 | 宏观新闻分析（ETF、美联储、监管政策） | CryptoPanic 新闻 API | 只报告有实质影响的事件 |
| 4 | 💎 **基本面分析师** | 💎 | 链上数据分析（交易所流向、巨鲸转账） | 链上数据工具 | 必须有数据支撑 |
| 5 | ⚖️ **研究经理** | ⚖️ | 综合所有分析师报告，做出综合判断 | 上述 4 份报告 | 指出一致性和分歧点 |
| 6 | 🛡️ **风险经理** | 🛡️ | 风险评估（仓位、杠杆、止损建议） | ATR、波动率数据 | 硬性风控规则 |
| 7 | 💼 **交易员** | 💼 | 交易建议（开仓/平仓方向） | 研究经理报告 + 风险经理建议 | 明确的方向和理由 |
| 8 | 🎯 **最终决策** | 🎯 | 最终交易决策（实际下单指令） | 交易员建议 + BTC 200 日均线拦截器 | 可执行的交易指令 |

## 飞书 Webhook 配置步骤

### 步骤 1：在飞书群中添加自定义机器人

1. 打开飞书，进入你想要接收通知的**群聊**
2. 点击右上角 **群设置** → **机器人** → **添加机器人**
3. 选择 **自定义机器人**
4. 填写机器人名称（例如："📈 市场分析师"）
5. 复制生成的 **Webhook 地址**

### 步骤 2：为每个角色创建独立的机器人

**推荐方案：每个角色一个独立的群聊 + 机器人**

| 群聊名称 | 机器人名称 | Webhook 变量 |
|----------|------------|--------------|
| 📈 市场分析群 | 📈 市场分析师机器人 | `FEISHU_WEBHOOK_ANALYST` |
| 😊 情绪分析群 | 😊 情绪分析师机器人 | `FEISHU_WEBHOOK_SENTIMENT` |
| 📰 新闻分析群 | 📰 新闻分析师机器人 | `FEISHU_WEBHOOK_NEWS` |
| 💎 基本面分析群 | 💎 基本面分析师机器人 | `FEISHU_WEBHOOK_FUNDAMENTALS` |
| ⚖️ 研究决策群 | ⚖️ 研究经理机器人 | `FEISHU_WEBHOOK_MANAGER` |
| 🛡️ 风险管理群 | 🛡️ 风险经理机器人 | `FEISHU_WEBHOOK_RISK` |
| 🎯 交易决策群 | 🎯 交易员机器人 | `FEISHU_WEBHOOK_TRADER` |

**简化方案：所有角色通知到同一个群**

如果不想创建多个群，可以：
- 只配置 `FEISHU_WEBHOOK_URL`（通用通知）
- 所有角色的通知会合并发送到同一个群

### 步骤 3：配置 .env 文件


## 提示词优化建议

### 当前问题

1. **部分角色提示词缺少数据验证要求** - AI 可能"瞎编"数据
2. **分析过程不透明** - 飞书通知只显示结论，没有显示推理过程
3. **缺少证据引用** - 没有强制要求 AI 引用具体数据来源

### 优化方案

#### ✅ 已实现的优化

1. **语言强制** - 通过 `get_language_instruction()` 强制中文输出
2. **结构化输出** - 所有分析师必须输出 JSON 格式，包含 `evidence` 字段
3. **数据门卫** - Python 先计算客观信号，再让 LLM 评估

#### 🔧 待实现的优化

**建议 1：为每个角色添加"分析过程显示"要求**

在每个角色的提示词中添加：

```python
"""
**分析过程透明化要求：**
1. 必须列出你使用的每一个数据点及其来源
2. 必须展示你的推理链条（从数据 → 信号 → 结论）
3. 必须标注数据的时间戳和完整性
4. 如果数据不完整，必须明确指出
"""
```

**建议 2：添加"反幻觉"指令**

```python
"""
**反幻觉规则（必须遵守）：**
1. 禁止编造任何数据 - 如果数据不可用，明确写"数据不可用"
2. 禁止模糊表述 - 不能使用"可能"、"也许"等词汇，必须有数据支撑
3. 禁止超出你的知识范围 - 只分析你被分配的职责
4. 所有数值必须来自工具调用，不能估算
"""
```

**建议 3：添加"证据引用"格式要求**

```python
"""
**证据引用格式：**
每个结论后面必须用方括号标注数据来源，例如：
- "RSI=35，显示超卖 [来源：技术指标计算]"
- "资金费率 +0.05%，多头支付费用 [来源：Bitget 资金费率 API]"
- "过去 24 小时交易所流入 5000 BTC [来源：链上数据]"
"""
```

## 完整配置示例

### .env 文件完整配置

```bash
# =============================================================================
# 飞书机器人 Webhook 配置
# =============================================================================

# 通用通知 Webhook（可选，作为后备通知渠道）
FEISHU_WEBHOOK_URL=https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxxxxxxxxxxxxxxxxxx

# 📈 市场分析师 Webhook（技术分析结果推送）
FEISHU_WEBHOOK_ANALYST=https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxxxxxxxxxxxxxxxxxx

# 😊 情绪分析师 Webhook（情绪分析结果推送，可选）
FEISHU_WEBHOOK_SENTIMENT=https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxxxxxxxxxxxxxxxxxx

# 📰 新闻分析师 Webhook（新闻分析结果推送，可选）
FEISHU_WEBHOOK_NEWS=https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxxxxxxxxxxxxxxxxxx

# 💎 基本面分析师 Webhook（基本面/链上数据分析推送，可选）
FEISHU_WEBHOOK_FUNDAMENTALS=https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxxxxxxxxxxxxxxxxxx

# ⚖️ 研究经理 Webhook（研究经理裁决推送）
FEISHU_WEBHOOK_MANAGER=https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxxxxxxxxxxxxxxxxxx

# 🛡️ 风险经理 Webhook（风险评估推送）
FEISHU_WEBHOOK_RISK=https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxxxxxxxxxxxxxxxxxx

# 🎯 交易员 Webhook（最终交易决策推送）
FEISHU_WEBHOOK_TRADER=https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxxxxxxxxxxxxxxxxxx

# =============================================================================
# LLM 提供商 - 通义千问 (Qwen)
# =============================================================================
DASHSCOPE_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

# 推荐使用：qwen-max (深度分析) + qwen-plus (快速决策)
# =============================================================================
# Bitget 交易所 API（永续合约交易）
# =============================================================================
BITGET_API_KEY=bg_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
BITGET_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
BITGET_PASSPHRASE=your_passphrase_here

# true = 沙盒模式（推荐测试时使用），false = 实盘交易（谨慎！）
BITGET_SANDBOX=true

# =============================================================================
# 交易参数
# =============================================================================
# 每次交易使用的资金量（USDT）
CAPITAL_USDT=1000

# K 线周期设置：4h = 4 小时线（默认），1d = 日线
TIMEFRAME=4h

# =============================================================================
# 调试选项
# =============================================================================
# true = 开启 LangGraph 详细 debug 输出（显示每个 agent 的中间状态）
DEBUG=false
```
