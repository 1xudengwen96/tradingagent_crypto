# 加密货币多智能体合约交易系统 — 使用说明书

## 1. 系统简介

本系统是一个基于 LLM 多智能体架构的加密货币永续合约 AI 交易系统。

### 核心理念
- **多智能体协作**：4 位分析师提供不同维度分析 → 研究员辩论 → 交易员制定计划 → 风险辩论 → 组合管理者最终决策
- **LLM 只定方向，数学管仓位**：大模型仅输出做多/做空/平仓方向，仓位大小由 Python 硬编码数学模型计算
- **自动化飞书通知**：每次分析结果自动推送到飞书群，随时掌握交易动态
- **Web 可视化管理**：通过浏览器界面完成配置、监控、启停全流程

### 系统架构

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  Bitget 数据  │───▶│  AI 多智能体  │───▶│  BitgetExecutor│
│  (OHLCV/费率) │    │  分析决策     │    │  (下单执行)    │
└──────────────┘    └──────────────┘    └──────┬───────┘
                                               │
                                      ┌────────▼────────┐
                                      │  飞书 Webhook    │
                                      │  (消息推送)      │
                                      └─────────────────┘
                                              ▲
                                      ┌───────┴───────┐
                                      │  Web 管理界面  │
                                      │  (监控/配置)   │
                                      └───────────────┘
```

## 3. 快速开始

### 3.1 命令行模式

```python
from tradingagents.graph.crypto_trading_graph import CryptoTradingAgentsGraph

# 创建实例（不开启自动下单，仅分析）
graph = CryptoTradingAgentsGraph(
    debug=False,
    auto_execute=False,  # 先设为 False 看分析结果
)

# 运行分析
final_state, decision_text = graph.propagate("BTC/USDT:USDT")
print(decision_text)
```

### 3.2 自动执行交易

```python
# 开启自动下单（⚠️ 确保 sandbox_mode=False 时有真实资金风险意识）
graph = CryptoTradingAgentsGraph(
    auto_execute=True,   # 自动执行订单
    debug=True,          # 打印中间过程
)

final_state, decision_text, execution_result = graph.run("BTC/USDT:USDT")

if execution_result and execution_result.success:
    print(f"订单成功: {[o.get('id') for o in execution_result.orders]}")
else:
    print(f"订单失败: {execution_result.error}")
```

### 3.3 查看账户余额

```python
graph = CryptoTradingAgentsGraph(auto_execute=True)
balance = graph.fetch_account_balance()
print(f"总资金: {balance['total']} USDT")
print(f"可用: {balance['free']} USDT")
```

### 3.4 使用主入口脚本

```bash
# 运行一次分析（不下单）
python crypto_main.py --once --no-execute

# 运行一次分析并下单（沙盒）
python crypto_main.py --once

# 定时运行（默认每 4 小时）
python crypto_main.py

# 每 2 小时运行一次
python crypto_main.py --interval-hours 2

# 只分析指定币种
python crypto_main.py --once --no-execute --symbol BTC/USDT:USDT
```

1. 在飞书群中添加「自定义机器人」

3. 在「交易设置」页面粘贴 Webhook URL
4. 点击「测试飞书通知」，确认飞书群收到测试消息
5. 点击「保存设置」

1. 在「交易设置」页面选择新的分析间隔

3. 确保 Bitget API 密钥已切换为实盘密钥
4. 确保账户有足够 USDT 保证金
5. **建议先在虚拟盘充分测试后再切换**

## 5. 风控规则

### 5.1 ATR 仓位计算公式

系统不使用 LLM 决定仓位，而是用硬编码数学模型：

```
止损距离 = 1.5 × ATR14（真实波动幅度）
基础仓位 = (总资金 × 1%) / (止损距离 / 入场价)
调整后仓位 = 基础仓位 × min(max(信念分/10, 0.5), 1.0)
最终仓位 = min(调整后仓位, 总资金 × 30%)
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 单笔最大亏损 | 1% | 无论信念分多高，单笔最多亏总资金 1% |
| ATR 倍数 | 1.5× | 止损 = 1.5 倍 ATR |
| 止盈 1 | 2R | 止损距离的 2 倍（平仓 50%） |
| 止盈 2 | 3.5R | 止损距离的 3.5 倍（平仓剩余） |
| 单笔上限 | 30% | 单笔仓位不超过总资金 30% |
| LITE 缩放 | 50% | LONG-LITE/SHORT-LITE 时仓位减半 |

### 5.2 BTC 200 日均线拦截器

```
如果 BTC 价格 < BTC 200 日均线 且 LLM 方向为 LONG/LONG-LITE：
    强制覆盖方向为 CLOSE（只能平仓，禁止开多）
```

这是硬性规则，LLM 无法覆盖。

### 5.3 止损/止盈强制挂单

每笔开仓订单发出后，系统**必须**立即挂出：
- **止损单（Stop Market）**：触发价格到达止损价时市价平仓
- **止盈单 1（Limit）**：挂 50% 仓位在 2R 止盈
- **止盈单 2（Limit）**：挂剩余 50% 仓位在 3.5R 止盈

如果信号中缺少止损/止盈价格，系统会自动用 ATR 回退计算。

## 7. 飞书通知

### 7.1 通知内容

每次 AI 分析完成后，飞书群会收到一条交互式卡片消息：

```
📊 BTC/USDT:USDT 分析报告
━━━━━━━━━━━━━━━━

[AI 完整分析决策内容...]

执行状态：✅ 已下单 / ⏸️ 仅分析 / ❌ 下单失败
时间：2026-04-06 12:00 UTC
```

### 7.2 通知状态说明

| 状态 | 含义 |
|------|------|
| ✅ 已下单 | AI 分析后自动执行了交易，并列出订单 ID |
| ⏸️ 仅分析 | 仅做了分析，未执行交易（auto_execute=false） |
| ❌ 下单失败 | AI 决定下单但执行失败，显示错误原因 |

### 7.3 获取飞书 Webhook URL

1. 打开飞书群 → 点击群设置（右上角 ⚙️）
2. 找到「群机器人」→ 点击「添加机器人」
3. 选择「自定义机器人（通过 Webhook 接入）」
4. 设置机器人名称（如「交易助手」），获取 Webhook URL
5. 将 URL 粘贴到「交易设置」页面的飞书 Webhook 输入框

### 7.4 测试通知

1. 在「交易设置」页面填写 Webhook URL
2. 点击「测试飞书通知」按钮
3. 检查飞书群是否收到测试卡片消息
4. 如果收到说明连接成功，如果报错请检查 URL 是否正确

## 11. 安全注意事项

### 11.1 上线前检查清单

- [ ] 在 `sandbox_mode = True` 下测试至少 7 天
- [ ] 确认 Bitget API 密钥正确且有永续合约权限
- [ ] 理解 ATR 仓位计算逻辑
- [ ] 确认止损/止盈挂单正常执行
- [ ] 测试飞书通知是否正常推送
- [ ] 确认分析间隔符合预期

### 11.2 风险控制

| 风险 | 保护措施 |
|------|----------|
| 黑天鹅暴跌 | 止损单自动挂出 + BTC 200MA 拦截 |
| LLM 幻觉 | 仓位由数学模型计算，LLM 不碰数字 |
| 过度集中 | 单笔仓位上限 30% 总资金 |
| 连续亏损 | 单笔最多亏 1%，10 次连亏最多亏 10% |

### 11.3 密钥安全

- API 密钥保存在服务器端，不在前端明文展示
- Web 界面输入的密钥在保存时自动脱敏显示
- 建议定期更换 API 密钥

## 12. 常见问题

**Q: 飞书通知收不到？**  
A: 1) 检查 Webhook URL 是否正确（以 `https://open.feishu.cn` 开头）；2) 点击「测试飞书通知」验证；3) 确认飞书群机器人已启用且未被管理员禁用。

**Q: 修改了分析间隔但没生效？**  
A: 间隔修改保存在配置中，需要停止并重新启动 Bot 才能读取新配置。流程：保存设置 → 停止 Bot → 启动 Bot。

**Q: 实盘余额显示为 0？**  
A: 确认：1) 已切换到实盘模式（非沙盒）；2) 使用的是实盘 API 密钥（非沙盒密钥）；3) 实盘账户有 USDT 余额。

**Q: CryptoPanic 新闻拉取失败？**  
A: 无 API Key 时公共接口限速 60次/小时。建议在 [cryptopanic.com](https://cryptopanic.com/developers/api/) 免费申请 Key。

**Q: 如何查看 AI 的详细分析过程？**  
A: 查看 `crypto_results/{symbol}/logs/state_{timestamp}.json` 文件，包含所有智能体的完整输出。Web 界面的「日志」页也显示关键事件。
